# 温度缩放与归一化的实际意义

这段代码的意义非常深刻，涉及**统计力学、信息论和强化学习**的核心概念。

## 🔥 温度缩放

```python
outputs = np.log(probs) / self._temperature
```

### 统计力学角度：Boltzmann 分布

在统计力学中，粒子处于某个状态的概率遵循 **Boltzmann 分布**：

$$P(i) = \frac{e^{-E_i / (kT)}}{\sum_j e^{-E_j / (kT)}}$$

其中：
- $E_i$ = 状态 $i$ 的能量
- $T$ = 温度
- $k$ = Boltzmann 常数

**对应到代码**：
- `np.log(probs)` $\approx$ 负能量 $-E_i$（概率越高 = 能量越低 = 越优）
- `self._temperature` = 温度 $T$

### 温度的作用

| 温度 | 效果 | 物理意义 | MCTS 中的效果 |
|------|------|----------|--------------|
| $T \to 0$ | 只有能量最低的状态有概率 | 系统冻结，进入基态 | **纯贪婪**，只选最优动作 |
| $T \to \infty$ | 所有状态概率相等 | 完全随机，高能态也能达到 | **纯随机**，探索所有动作 |
| $T = 1$ | 原始概率分布 | 正常温度 | **原始模型概率** |

**代码体现**：
```python
probs ** (1.0 / self._temperature)  # T < 1 时放大差异，T > 1 时缩小差异
```

## 🎲 归一化

```python
if self._normalize:
    outputs -= np.log(np.exp(outputs).sum())  # Log-Softmax
```

### 信息论角度：从 Logits 到概率分布

这是 **Log-Softmax** 操作：

$$\text{log-softmax}(x_i) = \log\left(\frac{e^{x_i}}{\sum_j e^{x_j}}\right) = x_i - \log\left(\sum_j e^{x_j}\right)$$

**为什么要做这个？**

1. **确保概率和为 1**（概率分布的基本要求）
2. **在 log 空间操作**（避免数值下溢）
3. **强化学习需要有效的概率分布**（用于采样、梯度计算）

### 数值稳定性技巧

```python
outputs -= outputs.max()  # 关键技巧！
```

**问题**：直接计算 $\log(\sum e^{x_i})$ 会数值溢出

**解决**：减去最大值（不改变结果）
$$\log\left(\sum_i e^{x_i}\right) = \log\left(e^{\max} \sum_i e^{x_i - \max}\right) = \max + \log\left(\sum_i e^{x_i - \max}\right)$$

## 🎯 在 MCTS Policy 中的实际意义

### 为什么需要温度缩放？

```python
# 假设模型预测 3 个反应的概率
probs = [0.7, 0.2, 0.1]

# T = 0.5 (低温，更"贪婪")
outputs = probs ** (1/0.5) = probs ** 2 = [0.49, 0.04, 0.01]
# 归一化后 ≈ [0.91, 0.07, 0.02]  ← 更倾向于选择最优的

# T = 2.0 (高温，更"探索")
outputs = probs ** (1/2) = probs ** 0.5 = [0.84, 0.45, 0.32]
# 归一化后 ≈ [0.52, 0.28, 0.20]  ← 更均匀，增加探索
```

### MCTS 中的应用场景

| 场景 | 温度设置 | 原因 |
|------|----------|------|
| **训练初期** | $T > 1$ | 鼓励探索，发现新策略 |
| **训练后期** | $T < 1$ | 利用已学知识，追求最优 |
| **推理时** | $T \to 0$ | 确定性输出，选择最佳动作 |

**AlphaGo 的经典做法**：
- 训练早期：$T = 1$（探索）
- 训练后期：$T \to 0$（利用）
- 这段代码让 MCTS 可以灵活调整探索-利用平衡


## 💡 总结

这段代码实现了**温度控制的概率分布变换**：

1. **温度缩放**：控制探索-利用平衡（来自统计力学）
2. **归一化**：确保有效概率分布（数学要求）
3. **Log 空间操作**：避免数值问题（计算技巧）

**物理直觉**：
- **低温度** = 系统冻结在基态 = 坚持选最好的
- **高温度** = 热涨落强 = 多尝试不同的

**代码的价值**：让 MCTS 可以通过调整温度参数，在"贪婪利用"和"广泛探索"之间灵活切换，这是强化学习和搜索算法中的核心技巧！
